When requesting features or projects, you must pass Izanami technical ids instead of project or feature names.
This ensure resilience of your Izanami client (changing names won't break queries), however it makes building query urls more complicated.

To help you building these urls, Izanami provides a **query builder**.

<img src={menuItemHover} />

This page allows to select several elements that will be included in the generated URL :

- Features : your query will return activation status for each selected feature
- Projects : your query will return activation status for **each feature of each selected projects**
- Context : your query will return activation statuses for given context
- User : your query will return activation statuses for given user

One you've got everything you need, you can either copy generated url, or make sure that it works as expected by testing it.

<img src={testResult} />

Result can also be displayed as JSON to help you process response in your application.
