title: Global VS local contexts
---

Izanami allows to create both local and global contexts.

Global contexts are bind to tenant, while local contexts are bind to a single project.

Basically, global context are usefull for setting up environnements, or other stuff that is shared for all tenant projects.

Local contexts on the other hand, allows to define specific project stuff.
